# 주식 스크리닝 프로젝트

이 파일은 Codex가 이 프로젝트에서 따라야 할 최상위 작업 지침이다.
역할별 상세 책임은 `.codex/agents/*.toml`을 기준으로 한다.

## 먼저 읽을 문서
- `CLAUDE.md`: 프로젝트 구조, 필터 정책, 배포 및 통합 규칙
- `MEMORY.md`: 기존 구현 결정과 알려진 제약
- `.claude/plans/PLAN.md`: 단계별 계획과 완료 상태
- `.codex/agents/*.toml`: Codex 서브에이전트 역할 정의

## 프로젝트 개요
- Python/Streamlit 기반 개인용 주식 스크리닝 앱
- 미국주식과 한국주식의 상대강도(RS) 순위를 한 화면에 표시
- 메인 화면은 미국주식 순위와 한국주식 순위를 좌우에 배치
- 종목 선택 시 해당 순위 행 바로 아래에 차트 표시
- 미국 데이터는 yfinance/FDR, 한국 데이터는 FinanceDataReader/LS증권 OpenAPI 사용
- SQLite 캐시와 GitHub Actions 원격 캐시 갱신 사용

## 에이전트 구성
- `agent1-manager`: 전체 작업 분배, 통합, 검토
- `agent2-frontend`: Streamlit UI, 레이아웃, 차트, 테마
- `agent3-backend`: RS 계산, 필터, 랭킹, 캐시, 배치 로직
- `agent4-us-data`: 미국주식 외부 데이터 API
- `agent5-kr-data`: 한국주식 외부 데이터 및 위험종목 API

## Codex 멀티에이전트 운영 규칙
- 사용자가 서브에이전트 사용, 역할 분담, 병렬 작업을 명시하면 위 역할 정의를 사용한다.
- 메인 에이전트는 기본적으로 `agent1-manager` 역할을 맡는다.
- 여러 영역 작업은 파일 소유권이 겹치지 않도록 나누고 가능한 작업만 병렬 실행한다.
- 역할 배정 시 해당 `.codex/agents/*.toml` 내용을 작업 지침으로 전달한다.
- 서브에이전트는 다른 작업자의 변경을 되돌리지 않으며 담당 파일만 수정한다.
- 메인 에이전트는 결과를 직접 검토하고 전체 테스트 후 최종 보고한다.
- 단순 수정은 불필요하게 분산하지 않고 메인 에이전트가 직접 처리한다.

## 업무 분담 기준
- UI/디자인/화면 출력/차트 렌더링 -> `agent2-frontend`
- RS 계산/필터/랭킹/SQLite 캐시/통계 -> `agent3-backend`
- yfinance/FDR 미국 종목 목록/미국 시세/미국 메타 -> `agent4-us-data`
- FinanceDataReader 한국 종목 목록/한국 시세/LS 위험종목 -> `agent5-kr-data`
- 여러 영역에 걸친 작업의 순서와 통합 -> `agent1-manager`

## 공통 작업 원칙
- 요청과 직접 관련 없는 리팩터링이나 포맷 변경을 하지 않는다.
- 기존 코드의 spec dict 및 공통 함수 패턴을 우선 재사용한다.
- 모든 `st.session_state` 키는 미국 `scr_`, 한국 `scr_kr_` 네임스페이스를 유지한다.
- 상승은 빨간색 `#ff4b4b`, 하락은 파란색 `#3182f6` 계열을 유지한다.
- 실제 사용자 DB와 캐시 파일을 삭제하거나 커밋하지 않는다.
- API 키, 비밀번호, `.env`, secrets 파일을 커밋하지 않는다.

## 검증 및 완료
- Python 변경 후 최소한 `py_compile`과 관련 테스트를 실행한다.
- 공통 로직이나 UI 흐름 변경 시 가능하면 전체 테스트를 실행한다.
- 작업 완료 후 변경 파일, 테스트 결과, 커밋 및 푸시 상태를 명확히 보고한다.

## 배포
- Streamlit Cloud는 GitHub `main` 브랜치를 읽는다.
- 로컬 수정만으로 배포되지 않는다.
- 작업 트리에 다른 사용자 변경이 있으면 해당 변경을 커밋에 포함하지 않는다.
